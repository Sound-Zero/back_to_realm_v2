# Open Source Checklist

This repository includes the core files GitHub expects for a healthy open-source project:

- README: project purpose, setup, commands, and layout
- LICENSE: MIT license
- CONTRIBUTING: contribution workflow
- CODE_OF_CONDUCT: contributor behavior expectations
- SECURITY: private vulnerability reporting guidance
- CHANGELOG: release history
- Issue templates: structured bug reports and feature requests
- Pull request template: review and validation checklist
- CI workflow: lint and test on supported Python versions
- `.gitignore`: excludes checkpoints, logs, caches, and generated metrics

Before publishing, add repository URLs to `pyproject.toml` once the GitHub owner/name is known.
